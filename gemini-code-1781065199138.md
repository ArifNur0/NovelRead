# Blueprint & Rencana Pengembangan: Web Koleksi Novel PDF

Proyek ini adalah aplikasi web berbasis Python untuk mengelola dan membaca koleksi novel pribadi dalam format PDF. Database menggunakan MySQL yang dikelola melalui XAMPP. Website ini mengusung desain minimalis, modern, dan *user-friendly* (berfokus pada visual cover buku).

---

## 1. Arsitektur & Spesifikasi Teknologi

* **Backend & API:** Python (Direkomendasikan menggunakan **Flask** karena ringan dan mudah mengintegrasikan HTML + API, atau **FastAPI** jika ingin fokus pada API murni).
* **Database:** MySQL (via XAMPP / phpMyAdmin).
* **Frontend:** HTML5, CSS3 (menggunakan **Tailwind CSS** untuk desain minimalis dan responsif), serta JavaScript (Fetch API) untuk interaksi dinamis.
* **Penyimpanan File:** * File PDF & Cover Buku disimpan di dalam *file system* server (folder lokal proyek).
    * Database hanya menyimpan *path/string* lokasi file tersebut.

---

## 2. Perancangan Database (MySQL)

Kita akan membuat database bernama `koleksi_novel` dengan dua tabel utama: `users` (untuk autentikasi) dan `novels` (untuk data novel).

### Tabel: `users`
| Kolom | Tipe Data | Atribut | Keterangan |
| :--- | :--- | :--- | :--- |
| `id` | INT | Primary Key, Auto Increment | ID Pengguna |
| `username` | VARCHAR(50) | Unique, Not Null | Username untuk login |
| `password` | VARCHAR(255) | Not Null | Password yang sudah di-hash (Bcrypt/Werkzeug) |

### Tabel: `novels`
| Kolom | Tipe Data | Atribut | Keterangan |
| :--- | :--- | :--- | :--- |
| `id` | INT | Primary Key, Auto Increment | ID Novel |
| `title` | VARCHAR(255) | Not Null | Judul Novel |
| `author` | VARCHAR(100) | | Nama Penulis |
| `cover_path` | VARCHAR(255) | Not Null | Jalur file gambar cover (e.g., `/static/covers/book1.jpg`) |
| `pdf_path` | VARCHAR(255) | Not Null | Jalur file PDF (e.g., `/static/pdfs/book1.pdf`) |
| `uploaded_at` | TIMESTAMP | Default CURRENT_TIMESTAMP | Waktu upload |
| `user_id` | INT | Foreign Key (`users.id`) | Mengidentifikasi siapa yang mengupload |
